# คู่มือใช้งาน (เริ่มตั้งแต่ login)

ระบบมี "login" 2 ระดับ:
1. **login บอท** — สิทธิ์ใช้บอท (คุมด้วย `ALLOWED_CHAT_IDS`)
2. **login เว็บ** — เชื่อม WordPress แต่ละเว็บเข้าระบบด้วย `/addsite` (ใส่ url + API key ของเว็บนั้น)

> 🔑 ระบบใช้แค่ **API key (X-Kim-Key)** ต่อเว็บ ไม่ใช้รหัส WordPress ของคุณ

---

## ก่อนเริ่ม (ทำครั้งเดียวต่อเว็บ)
ติดตั้งปลั๊กอิน `kim-mcp-bridge` บนเว็บ → Activate → Settings > Kim MCP → ตั้ง API key
**ถ้าเว็บนั้นลงปลั๊กอินไว้อยู่แล้ว ข้ามขั้นนี้ได้เลย** — ไปเชื่อมที่ขั้นตอน 2 ได้ทันที

---

## ลำดับคำสั่ง (สรุป)
```
/start                                  ← (1) login บอท: เช็คสิทธิ์ + ดูสถานะ
/addsite siteA https://a.com KEY123     ← (2) login เว็บ A: เชื่อมปลั๊กอิน (บอทเช็ค health ให้)
/use siteA                              ← (3) เลือกเว็บที่จะสั่งงาน
/report                                 ← (4) ลองอ่านข้อมูล
เขียนบทความเรื่อง... ลง siteA ใส่รูปด้วย   ← (5) สั่งงานจริง (ภาษาไทย)
```

---

## ทีละขั้น

### 1) login บอท — `/start` หรือ `/login`
เปิดแชทบอทใน Telegram แล้วพิมพ์ `/start`
- ถ้าตั้ง `ALLOWED_CHAT_IDS` ไว้ เฉพาะ chat id ที่อนุญาตเท่านั้นจะใช้ได้ (บอทจะบอก id ของคุณถ้าไม่มีสิทธิ์)
- บอทจะแสดงเว็บที่เชื่อมแล้ว + เว็บที่กำลังใช้

### 2) login เว็บ — `/addsite <ชื่อ> <url> <key>`
```
/addsite siteA https://a.com KEY123
```
- บอทบันทึกเข้าทะเบียน → reload MCP → **ยิง `/health` ตรวจให้อัตโนมัติ**
- สำเร็จจะเห็น: `✅ เชื่อมเว็บ "siteA" สำเร็จ` พร้อมเวอร์ชันปลั๊กอิน + SEO plugin ที่ตรวจพบ
- ⚠️ **ความปลอดภัย:** key จะอยู่ในประวัติแชท Telegram — แนะนำใช้แชทส่วนตัวกับบอท และลบข้อความ `/addsite` หลังเชื่อมเสร็จ (key ถูกบันทึกในไฟล์ฝั่งเซิร์ฟเวอร์แล้ว)

### 3) เลือกเว็บ — `/use <ชื่อ>`
```
/use siteA
```
ตั้งเป็นเว็บเริ่มต้นของแชทนี้ — คำสั่งถัดไป (และการพิมพ์ภาษาไทย) จะยิงไปเว็บนี้โดยไม่ต้องระบุชื่อซ้ำ

### 4–5) สั่งงาน
**อ่านข้อมูล** (ไม่ใส่ชื่อ = ใช้เว็บที่ /use ไว้):
```
/report          /health          /posts
```
**จัดการโพสต์:**
```
/publish 12      /draft 12        /delete 12
```
**เขียนบทความ / งานซับซ้อน (ภาษาไทย):**
```
เขียนบทความเรื่องวิธีชงกาแฟดริปลง siteA ใส่รูปสวยๆ ตั้งเป็น draft ก่อน
```
Claude จะเขียนเนื้อหา + SEO เอง, Gemini สร้างรูป, แล้วลงให้ตามที่สั่ง

---

## สั่งงานแยกแต่ละโดเมน
เชื่อมหลายเว็บไว้ แล้วทำงานแยกได้ 2 แบบ:

**แบบ A — สลับเว็บที่ใช้อยู่:**
```
/use siteA
/report            (→ siteA)
/use siteB
เขียนบทความ...      (→ siteB)
```

**แบบ B — ระบุชื่อเว็บในคำสั่งตรงๆ (ไม่ต้องสลับ):**
```
/report siteB
/publish siteA 12
"ลงบทความรีวิวร้านกาแฟที่ siteB"
```

## ติดตั้งปลั๊กอินอัตโนมัติ — `/onboard` (เว็บใหม่ที่ยังไม่ลงปลั๊กอิน)
ถ้าเว็บ**ยังไม่มีปลั๊กอิน** ใช้คำสั่งนี้ให้บอทติดตั้งให้อัตโนมัติ — ทำ **ทีละโดเมน**:
```
/onboard siteA https://a.com admin-user admin-pass
```
บอทจะ: login wp-admin → อัปโหลด+activate ปลั๊กอิน → สุ่ม+ตั้ง API key เอง → เชื่อมเข้าระบบ → เช็ค health → ตั้งเป็นเว็บที่ใช้อยู่ (จบในคำสั่งเดียว)

⚠️ **ข้อจำกัด/ความปลอดภัย:**
- ใช้ **รหัส admin จริง** — และจะลบข้อความ `/onboard` ทิ้งหลังเสร็จ (มีรหัสอยู่ในแชท)
- **ไม่ทำงาน** ถ้าเว็บมี 2FA / CAPTCHA / security plugin (Wordfence ฯลฯ) กัน login อัตโนมัติ หรือโฮสต์ปิดการอัปโหลดปลั๊กอิน → กรณีนั้นใช้วิธี manual (อัปเอง + `/addsite`)
- ใช้ cookie login (ไม่ใช่ Basic Auth) จึงไม่โดน cPanel ตัด header

> เปรียบเทียบ: `/onboard` = เว็บใหม่ (ลงปลั๊กอินให้) · `/addsite` = เว็บที่ลงปลั๊กอินแล้ว (ใส่แค่ key)

## เพิ่มเว็บโดยไม่ผ่าน Telegram
ทะเบียนเว็บคือไฟล์ `mcp-server/config/domains.json` เพิ่มได้ 2 ทาง:

**ก) CLI (สะดวก เพิ่มหลายเว็บรวด):**
```bash
cd mcp-server
node add-site.mjs add siteA https://a.com KEY_A
node add-site.mjs add siteB https://b.com KEY_B
node add-site.mjs list
node add-site.mjs remove siteA
```

**ข) แก้ไฟล์ตรงๆ:**
```json
{
  "siteA": { "url": "https://a.com", "key": "KEY_A" },
  "siteB": { "url": "https://b.com", "key": "KEY_B" }
}
```
> หลังเพิ่มด้วย CLI/แก้ไฟล์ ถ้าบอทรันอยู่แล้วให้ **รีสตาร์ทบอท** (หรือใช้ `/addsite` ในแชทแทน ซึ่ง reload อัตโนมัติ)

## คำสั่งทั้งหมด

| คำสั่ง | ทำอะไร |
|--------|--------|
| `/start` `/login` | login บอท: ดูสถานะ + เว็บที่เชื่อม |
| `/addsite <ชื่อ> <url> <key>` | เชื่อมเว็บที่ลงปลั๊กอินแล้ว (เช็ค health อัตโนมัติ) |
| `/onboard <ชื่อ> <url> <user> <pass>` | ติดตั้งปลั๊กอินอัตโนมัติให้เว็บใหม่ |
| `/removesite <ชื่อ>` | เอาเว็บออก |
| `/sites` | ดูเว็บทั้งหมด |
| `/use <ชื่อ>` | เลือกเว็บเริ่มต้นของแชท |
| `/health [ชื่อ]` | เช็คปลั๊กอิน |
| `/report [ชื่อ]` | สรุปภาพรวมเว็บ |
| `/posts [ชื่อ]` | โพสต์ล่าสุด |
| `/publish [ชื่อ] <id>` | เผยแพร่โพสต์ |
| `/draft [ชื่อ] <id>` | เปลี่ยนเป็น draft |
| `/delete [ชื่อ] <id>` | ลบโพสต์ |
| `/help` | คำสั่งทั้งหมด |
| _ภาษาไทย_ | เขียนบทความ/งานซับซ้อน → Claude + Gemini |

> `[ชื่อ]` = ใส่ก็ได้ ไม่ใส่ก็ได้ (ไม่ใส่ = ใช้เว็บที่ `/use` ไว้) · `<...>` = ต้องใส่
