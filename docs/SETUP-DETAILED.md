# คู่มือฉบับละเอียดสุด สำหรับมือใหม่ (ทำตามทีละขั้น)

> เขียนแบบไม่ต้องมีพื้นเทคนิคก็ทำได้ — อ่านจากบนลงล่าง ทำทีละข้อ ห้ามข้าม
> ถ้าติดตรงไหน ดูหัวข้อ **"ถ้าพังทำไง"** ท้ายสุด

---

## ส่วนที่ 0 — ระบบนี้คืออะไร (อ่าน 1 นาที)
- มี **บอท Telegram** ตัวนึง พิมพ์สั่งงานเว็บ WordPress ได้หลายเว็บ
- แต่ละเว็บต้องลง **ปลั๊กอินตัวเล็กๆ** (ชื่อ kim-mcp-bridge) ไว้ 1 ครั้ง เพื่อให้บอทสั่งงานได้
- เครื่องที่รันบอท = คอมของเรา/VPS (เปิดทิ้งไว้) — **ไม่ใช่** ที่โฮสต์ WordPress

```
เราพิมพ์ใน Telegram → บอท(บนคอมเรา) → ปลั๊กอินบนเว็บ → เว็บทำงานจริง
```

---

## ส่วนที่ 1 — เตรียมเครื่อง (ลงโปรแกรมพื้นฐาน)

ต้องมี 3 อย่างบนเครื่องที่จะรันบอท เปิดโปรแกรม **Terminal / PowerShell / Git Bash** แล้วพิมพ์ตามนี้

**1.1 เช็ค Node.js (ต้องเวอร์ชัน 20 ขึ้นไป)**
```bash
node -v
```
- ถ้าขึ้นเลข `v20.x` หรือมากกว่า = ผ่าน
- ถ้าขึ้น "command not found" → ไปโหลดที่ https://nodejs.org เลือกปุ่ม **LTS** ติดตั้งแบบกด Next ไปเรื่อยๆ แล้วเปิด Terminal ใหม่ ลอง `node -v` อีกครั้ง

**1.2 เช็ค Git**
```bash
git --version
```
- ไม่มี → โหลดที่ https://git-scm.com ติดตั้งแบบกด Next

**1.3 ลง pm2 (ตัวทำให้บอทรันค้างไม่ดับ)**
```bash
npm install -g pm2
```
- รอจนเสร็จ แล้วเช็ค: `pm2 -v` ต้องขึ้นเลขเวอร์ชัน

## ส่วนที่ 2 — ขอ "กุญแจ" 3 ดอก (สำคัญมาก ทำทีละที่)

### 2.1 Telegram Bot Token (รหัสบอท)
1. เปิดแอป Telegram → ช่องค้นหา พิมพ์ **@BotFather** → กดเข้าไป (มีติ๊กถูกสีฟ้า)
2. กด **Start** แล้วพิมพ์ `/newbot` ส่ง
3. มันถามชื่อบอท → พิมพ์ชื่ออะไรก็ได้ เช่น `เว็บของฉัน`
4. มันถาม username → พิมพ์ชื่อที่ **ลงท้ายด้วย bot** เช่น `myweb_ctrl_bot` (ห้ามซ้ำคนอื่น)
5. มันจะส่งข้อความมี **token** หน้าตาแบบ `123456789:AAE...xyz` → **ก็อปเก็บไว้** (นี่คือ TELEGRAM_BOT_TOKEN)

### 2.2 Anthropic API key (สำหรับให้ AI เขียนบทความ — ถ้าจะใช้)
1. เข้า https://console.anthropic.com → สมัคร/ล็อกอิน
2. เมนู **API Keys** → **Create Key** → ตั้งชื่อ → **Copy** เก็บไว้ (ขึ้นต้น `sk-ant-...`)
> ต้องเติมเงิน/มีเครดิตในบัญชีถึงจะใช้ได้

### 2.3 Gemini API key (สำหรับสร้างรูป — ถ้าจะใช้ และใช้ตัวเดียวกับ slip-bot ได้)
1. เข้า https://aistudio.google.com/apikey → ล็อกอิน Google
2. กด **Create API key** → **Copy** เก็บไว้ (ขึ้นต้น `AIza...`)

> 💡 ถ้าพี่ใช้ **ปลั๊กอิน workflow เดิม** สร้างบทความ/รูปเองอยู่แล้ว — ข้อ 2.2 และ 2.3 **ยังไม่จำเป็นตอนนี้** (ดูส่วนที่ 8)

## ส่วนที่ 3 — เอาโค้ดลงเครื่อง
ใน Terminal พิมพ์:
```bash
git clone https://github.com/adminfizz/wp-mcp.git
cd wp-mcp
```
ตอนนี้เราอยู่ในโฟลเดอร์ `wp-mcp` แล้ว

## ส่วนที่ 4 — ลงปลั๊กอินบน WordPress (ทำต่อเว็บ)

> ปลั๊กอินตัวนี้คือ "ตัวกลาง" ให้บอทสั่งงานเว็บได้ มี 2 วิธี — **มือใหม่แนะนำวิธี A (ทำมือ)**

### วิธี A — ทำมือ (ชัวร์ทุกเว็บ)
**4.1 บีบไฟล์ปลั๊กอินเป็น zip**
- เปิดโฟลเดอร์ `wp-mcp/wordpress-plugin/`
- คลิกขวาที่โฟลเดอร์ **kim-mcp-bridge** → (Windows) **Send to → Compressed (zipped) folder**
- ได้ไฟล์ `kim-mcp-bridge.zip`

**4.2 อัปขึ้น WordPress**
1. เข้าหน้าจัดการเว็บ: `https://เว็บของคุณ/wp-admin` → ล็อกอิน
2. เมนูซ้าย **Plugins (ปลั๊กอิน)** → **Add New (เพิ่มใหม่)** → ปุ่ม **Upload Plugin (อัปโหลด)**
3. **Choose File** → เลือก `kim-mcp-bridge.zip` → **Install Now**
4. กด **Activate Plugin (เปิดใช้งาน)**

**4.3 ตั้งรหัส API key ของเว็บนี้**
1. เมนูซ้าย **Settings (ตั้งค่า)** → **Kim MCP**
2. ช่อง **API Key** → พิมพ์รหัสยาวๆ มั่วๆ ของเราเอง เช่น `mySite123secretKEY` → **Save Changes**
3. **จดรหัสนี้ไว้** (เดี๋ยวเอาไปใส่ในบอท) — ตั้งคนละรหัสต่อเว็บยิ่งดี

### วิธี B — ให้บอทลงให้อัตโนมัติ
ข้ามส่วนนี้ไปก่อน แล้วใช้คำสั่ง `/onboard` ในส่วนที่ 7 (บอทจะ login เข้า wp-admin ลงปลั๊กอิน+ตั้งรหัสให้เอง) — ใช้ได้ถ้าเว็บไม่มี 2FA/ระบบกันบอท

## ส่วนที่ 5 — ตั้งค่าฝั่งคอมเรา (MCP + บอท)

### 5.1 ตั้งค่า MCP (ตัวเชื่อมไปเว็บ)
ใน Terminal (ยังอยู่ในโฟลเดอร์ wp-mcp):
```bash
cd mcp-server
npm install
```
รอจนเสร็จ (อาจ 1-2 นาที) แล้วสร้างไฟล์รายชื่อเว็บ:
```bash
cp config/domains.example.json config/domains.json
```
ใส่เว็บแรกด้วยคำสั่ง (เปลี่ยนค่าให้ตรงของจริง — `siteA` ตั้งชื่อเล่นเอง, ตามด้วย url, ตามด้วยรหัสจากข้อ 4.3):
```bash
node add-site.mjs add siteA https://เว็บของคุณ.com mySite123secretKEY
```
เช็คว่าเข้าแล้ว:
```bash
node add-site.mjs list
```

### 5.2 ตั้งค่าบอท
```bash
cd ../telegram-bot
npm install
cp .env.example .env
```
เปิดไฟล์ `.env` ด้วย Notepad (หรือ `notepad .env`) แล้วเติมค่า — บรรทัดไหนไม่ใช้เว้นว่างได้:
```ini
TELEGRAM_BOT_TOKEN=วาง token จากข้อ 2.1 ตรงนี้
ANTHROPIC_API_KEY=วาง key จากข้อ 2.2 (ถ้าจะใช้ AI เขียน)
GEMINI_API_KEY=วาง key จากข้อ 2.3 (ถ้าจะใช้สร้างรูป)
ALLOWED_CHAT_IDS=
```
**Save** ปิดไฟล์

## ส่วนที่ 6 — เปิดบอท

```bash
pm2 start ecosystem.config.js
pm2 logs wp-mcp-bot
```
- ถ้าเห็นข้อความ **"wp-mcp-bot พร้อมทำงาน ✅"** = สำเร็จ! (กด Ctrl+C เพื่อออกจากหน้า log บอทยังรันอยู่)
- สั่งให้บอทรันต่อแม้รีสตาร์ทเครื่อง: `pm2 save` แล้วทำตามที่ `pm2 startup` บอก

### ตั้งให้เฉพาะเราสั่งบอทได้ (ความปลอดภัย)
1. เปิดแชทบอทของเราใน Telegram → พิมพ์อะไรก็ได้ส่งไป
2. บอทจะตอบ **"...chat นี้ไม่มีสิทธิ์ (id: 123456789)"** → ก็อปเลข id นั้น
3. ใส่ลงไฟล์ `.env` บรรทัด `ALLOWED_CHAT_IDS=123456789` → Save
4. รีสตาร์ทบอท: `pm2 restart wp-mcp-bot`

## ส่วนที่ 7 — ลองใช้งานจริง (พิมพ์ใน Telegram)

```
/start                  → บอททักทาย + โชว์เว็บที่เชื่อม
/sites                  → ดูเว็บทั้งหมด
/use siteA              → เลือกว่าจะสั่งงานเว็บ siteA
/status                 → 🟢 ออนไลน์ / 🔴 หลุด ของแต่ละเว็บ
/report                 → สรุปจำนวนโพสต์ของ siteA
/log siteA              → ดูว่า siteA ทำอะไรไปบ้าง (เวลา+งาน)
```
เพิ่มเว็บใหม่ทีหลัง (2 วิธี):
```
/addsite siteB https://b.com รหัสของB       ← เว็บที่ลงปลั๊กอินแล้ว
/onboard siteC https://c.com แอดมิน รหัสแอดมิน  ← ให้บอทลงปลั๊กอินให้อัตโนมัติ
```
> ลบข้อความ `/onboard` ทิ้งหลังใช้เสร็จ เพราะมีรหัสแอดมินอยู่

## ส่วนที่ 8 — เชื่อมกับ "ปลั๊กอินอื่น" ที่มีอยู่แล้วบนเว็บ

> เช่น ปลั๊กอิน workflow ที่สร้างบทความเอง, WooCommerce, ปลั๊กอินแคช ฯลฯ
> ให้สั่งงานมันผ่าน Telegram รายเว็บได้ — มี 2 กรณี

### กรณี 1 — ปลั๊กอินนั้นมี REST API ของตัวเอง
รวบรวมข้อมูลพวกนี้จากปลั๊กอินนั้น (ดูในเอกสารปลั๊กอิน หรือถามคนทำ):
- **URL ของคำสั่ง** เช่น `https://a.com/wp-json/ชื่อปลั๊กอิน/v1/run`
- **ต้องแนบรหัสยังไง** เช่น header `Authorization: Bearer xxx` หรือ `?api_key=xxx`
- **ส่งข้อมูลอะไรไป** เช่น `{"topic":"หัวข้อบทความ"}`
- **มันตอบอะไรกลับ** (ตัวอย่าง response)

ส่งข้อมูล 4 อย่างนี้ให้ผู้ดูแลระบบ (หรือเลขาคิม) → จะเพิ่มคำสั่งให้สั่งตรงไปยัง API นั้นรายโดเมน
> ส่วนนี้กำลังพัฒนาให้ตั้งค่าผ่านไฟล์ได้เอง (ดู ROADMAP)

### กรณี 2 — ปลั๊กอินนั้นไม่มี API (กดผ่านปุ่มเท่านั้น)
ใช้ "ช่องเสียบ" ของ kim-mcp-bridge:
1. เปิดไฟล์ของเว็บนั้น (ผ่าน wp-admin → **Tools/เครื่องมือ** ของธีม หรือใช้ปลั๊กอินใส่โค้ดอย่าง "Code Snippets") เพิ่มโค้ดนี้:
   ```php
   add_filter('kim_mcp_action_clear_cache', function ($res, $payload) {
       if (function_exists('wp_cache_clear_cache')) wp_cache_clear_cache();
       return ['cleared' => true];
   }, 10, 2);
   ```
   (เปลี่ยน `clear_cache` เป็นชื่ออะไรก็ได้ และเปลี่ยนข้างในเป็นฟังก์ชันของปลั๊กอินที่อยากเรียก)
2. สั่งผ่าน Telegram:
   ```
   /action siteA clear_cache
   /action siteA run_workflow {"topic":"วิธีชงกาแฟ"}
   ```
> รายละเอียดเพิ่มเติม + ตัวอย่าง WooCommerce ดูที่ [CONNECT-EXISTING.md](./CONNECT-EXISTING.md)

## ส่วนที่ 9 — ถ้าพังทำไง (แก้ปัญหาที่เจอบ่อย)

| อาการ | ทำยังไง |
|-------|---------|
| `node -v` / `git` / `pm2` ขึ้น "not found" | ยังไม่ได้ลง/ลงไม่เสร็จ → กลับไปส่วนที่ 1 ปิด-เปิด Terminal ใหม่ |
| `npm install` แดงเต็มจอ | เน็ตหลุด/ติด proxy → ลองใหม่ `npm install` อีกรอบ |
| บอทไม่ตอบใน Telegram | ดู `pm2 logs wp-mcp-bot` มีข้อความแดงไหม · เช็ค `ALLOWED_CHAT_IDS` ใส่ id ถูกไหม · token ถูกไหม |
| `/status` ขึ้น 🔴 auth หลุด/key ผิด | รหัสในบอท (ข้อ 5.1) ไม่ตรงกับที่ตั้งบนเว็บ (ข้อ 4.3) → ตั้งใหม่ให้ตรงกัน |
| `/status` ขึ้น 🔴 เข้าไม่ถึงเว็บ | url ผิด หรือเว็บล่ม → เช็ค url พิมพ์ถูกไหม เปิดเว็บในเบราว์เซอร์ได้ไหม |
| `/onboard` ไม่สำเร็จ | เว็บมีระบบกันบอท (2FA/CAPTCHA/security plugin) → ใช้วิธี A (ทำมือ) แทน |
| อยากแก้ค่าแล้วให้มีผล | แก้ไฟล์เสร็จ → `pm2 restart wp-mcp-bot` |

**คำสั่ง pm2 ที่ใช้บ่อย**
```bash
pm2 logs wp-mcp-bot     # ดูว่าบอทพิมพ์อะไร (ดู error)
pm2 restart wp-mcp-bot  # รีสตาร์ทหลังแก้ค่า
pm2 stop wp-mcp-bot     # หยุดบอท
pm2 status              # ดูว่าบอทรันอยู่ไหม
```
